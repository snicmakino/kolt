**New**

- `kolt remove <group:artifact>` — drop a dependency from `kolt.toml`. Auto-searches both `[dependencies]` and `[test-dependencies]` (#343).

**Breaking — CLI changes**

- `kolt deps` namespace removed. Use the top-level commands directly: `kolt add` / `kolt update` / `kolt tree` (#339).
- `kolt install` renamed to `kolt fetch`. `kolt toolchain install` keeps its name (#339).
- `kolt check --watch` removed. Wire an LSP for live type-check feedback; `--watch` remains valid for `build` / `run` / `test` (#340).

**Behavior**

- `kolt update` no longer deletes cached JARs. Two consequences for users: warm-cache updates skip re-download (faster), and `git checkout` to an older `kolt.lock` builds without network if the cache hits. Trade-off: stale or corrupt cache (e.g. from a killed `kolt fetch`) must now be cleared explicitly via `kolt cache clean` — `kolt update` will not re-download to "fix" it (#347).

**Installation**

```sh
curl -fsSL https://raw.githubusercontent.com/snicmakino/kolt/main/install.sh | sh
```

**Upgrade notes**

- Replace `kolt deps add <gav>` with `kolt add <gav>`, `kolt deps update` with `kolt update`, `kolt deps tree` with `kolt tree`.
- Replace `kolt install` (and `kolt deps install`) with `kolt fetch`.
- Drop `--watch` from any `kolt check` invocations in scripts / CI.
