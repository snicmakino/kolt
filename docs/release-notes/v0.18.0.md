**`kolt update` no longer evicts cached JARs (#347)**

`kolt update` re-resolves and rewrites `kolt.lock` but no longer deletes JARs from `~/.kolt/cache/`. The cache is host-shared across projects, and a project-local `update` invalidating other projects' cache entries was a bad cross-project side effect — peer tools (cargo, npm, yarn, pnpm, poetry) all leave the shared cache alone. Side benefit: `git checkout` to an older `kolt.lock` after `update` no longer forces a re-download. Use `kolt cache clean` for explicit eviction.

**`kolt remove <group:artifact>` (#343)**

Closes the per-invocation symmetry gap named in ADR 0033 §7. Auto-searches both `[dependencies]` and `[test-dependencies]`; if the same coord lives in both, both are dropped. Removing an absent entry is idempotent (exit 0 with a one-line message). The shared cache at `~/.kolt/cache/` is left untouched — use `kolt cache clean` to reclaim disk space.

**`kolt deps` namespace dropped, `install` -> `fetch` (#339) — breaking**

The `kolt deps` namespace is removed. `add`, `update`, and `tree` were already top-level aliases; they are now the only spelling. The former `kolt install` (and `kolt deps install`) is renamed to `kolt fetch` — `install` reads as "install something on my system" but the command only downloads jars. `kolt toolchain install` keeps the noun because it really does install a tool.

Pre-v1, no compat shim. `kolt deps <anything>` and `kolt install` exit as unknown commands.

**`kolt check --watch` removed (#340) — breaking**

`kolt check --watch` overlapped with LSP responsibility. The flag is now rejected with a one-line error pointing at LSP; `--watch` remains valid for `build`, `run`, and `test`.

**Installation**

```sh
curl -fsSL https://raw.githubusercontent.com/snicmakino/kolt/main/install.sh | sh
```

**Upgrade notes**

- Replace `kolt deps add <gav>` with `kolt add <gav>`, `kolt deps update` with `kolt update`, `kolt deps tree` with `kolt tree`.
- Replace `kolt install` (and `kolt deps install`) with `kolt fetch`.
- Drop `--watch` from any `kolt check` invocations in scripts / CI. `kolt check` is fast enough for one-shot use; for live editor feedback, wire an LSP.
