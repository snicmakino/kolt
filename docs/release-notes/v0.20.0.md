**New**

- `[repositories.<name>]` accepts authenticated Maven repositories via `token = { literal = "..." }` (HTTP Bearer) or paired `user = { literal = "..." }` + `password = { literal = "..." }` (HTTP Basic). Declare credential values in `kolt.local.toml`; `kolt.toml` is rejected at parse time if it carries any of these fields. (#416)
- `kolt.local.toml` is read alongside `kolt.toml` as a per-user overlay for environment-specific values. The overlay can extend or override sys_props and repository fields from `kolt.toml` but cannot introduce repositories or tools that the base file does not declare. (#320, #415)
- `kolt init` appends `kolt.local.toml` to `.gitignore` automatically when it creates or updates the file. (#417)

**Behavior**

- The legacy `[repositories]` flat-string-map form (`central = "https://..."`) is no longer accepted. Migrate each entry to the sub-table form: `[repositories.<name>]` followed by `url = "..."`. The parser emits a hint pointing at the new shape. (#320)
- A 401 or 403 response from any repository stops resolution immediately and prints a diagnostic naming the repository, the redacted URL, the status code, the configured credential state, and a remediation hint. 404 fall-through to the next repository is unchanged. (#416)
- URLs containing user info (`https://user:pass@host/...`) are rejected at parse time; declare credentials in the dedicated `token` / `user` / `password` fields instead. (#416)

```sh
curl -fsSL https://raw.githubusercontent.com/snicmakino/kolt/main/install.sh | sh
```
