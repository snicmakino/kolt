**Bugfix**

- `kolt run` no longer aborts at runtime with `IrLinkageError` on a Kotlin/Native build that depends on a library publishing cinterop sub-klibs (notably ktor 3.4.3 `ktor-io`, `ktor-network`, `ktor-server-core`, `ktor-utils`). The resolver now fetches every `.klib` in each Gradle module variant's `files[]` rather than just the first, so manifest-declared cinterop dependencies link in (#430).

**Installation**

```sh
curl -fsSL https://raw.githubusercontent.com/snicmakino/kolt/main/install.sh | sh
```
