**Bugfix**

- `kolt add` no longer aborts on a Kotlin/Native build when a transitive dependency lacks Gradle Module Metadata. Affected `kolt add io.ktor:ktor-server-core` and similar mixed-platform libraries pulling JVM-only Kotlin artifacts (e.g. `kotlin-reflect`) into a `target = "linuxX64"` project; such transitives are now skipped with a single stderr note while the build continues (#426).

**Installation**

```sh
curl -fsSL https://raw.githubusercontent.com/snicmakino/kolt/main/install.sh | sh
```
